Code behind mode: code logic is separated from the view.

Account folder contains forms associated with authorization logic.
App_Data contains the local database files.
Content contains client css files.
fonts => fonts... thanks captain Obvious!
Models => Identity models.

.aspx files are the actual pages.

favicon - the default icon.

bundle.config -> at the moment: bundle all the css.

Global.asax =>  create bundles and routes.

packages.config ->  all the extra stuff the application needs to run or develop.

Master pages =>  shared things for all pages, lake navigation, styles, scripts.
StartUp => probably start up.
ViewSwitcher => helps with resolution issues.
web.config => more configurations, database connection string for example
